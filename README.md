# 🤖 노코드 자동화 도구 비교 및 파이프라인 설계 미션

## 📌 1. 미션 소개

매일 아침 이메일을 확인하고 스프레드시트에 옮겨 적고 팀 채널에 요약을 공유하는 루틴을 떠올려 봅시다.  
한 번에 5분이면 끝나는 일이지만 하루에 열 번 반복되면 어느새 **50분이 사라집니다.**

> "이걸 자동으로 돌릴 수 없을까?"

**노코드 자동화 도구**는 바로 이 지점에서 출발합니다.  
Make나 Zapier 같은 플랫폼은 코딩 없이 **Trigger(시작 이벤트)** 와 **Action(처리 동작)** 을  
시각적으로 연결해 반복 업무를 자동화합니다.

### 🎯 이번 미션에서는
- 동일한 워크플로우를 **서로 다른 2개 이상의 자동화 도구**로 구현
- 도구별 **비교 분석** 진행
- **자유 주제**로 나만의 자동화 파이프라인 설계 및 구현

> 단순히 "돌아가기만 하면 된다"가 아니라  
> Trigger와 Action의 **작동 원리를 이해**하고, **조건 분기를 설계**하며,  
> **도구별 장단점을 근거 있게 비교**하는 것이 핵심입니다.

## ⚙️ 4. 기능 요구 사항

### 공통 요구 사항
✅ Trigger 1개 이상 포함
✅ Action 2개 이상 포함
✅ 조건 분기(Filter/Router) 1개 이상 포함
✅ 각 분기 경로가 실제로 1회 이상 실행된 결과 확인 가능


### 프로젝트 1 요구 사항

- 서로 다른 **2개 이상**의 자동화 도구 사용
- **동일한 워크플로우 구조**로 구현
- 비교 분석 보고서 필수 항목:

| 항목 | 설명 |
|------|------|
| 사용한 도구 이름 | 비교 대상 도구 명시 |
| 구현 과정 요약 | 각 도구별 구현 절차 |
| 비교 항목 | **최소 5개 이상** (UI/UX, 설정 난이도, 연동 범위, 무료 플랜, 실행 로그 등) |
| 장단점 정리 | 각 도구의 장점 / 단점 |
| 적합한 상황 | 어떤 상황에서 어떤 도구가 적합한지 의견 |

### 프로젝트 2 요구 사항

- 자동화할 반복 업무 **1개** 정의
- 도구 **1개** 선정 + 선정 이유 작성
- **자동 실행 구조** 구현
- 워크플로우 **흐름 설명** 포함

## ⭐ 5. 보너스 과제 (선택)

### 보너스 1 – AI 연동 Action 추가
> 워크플로우에 생성형 AI(ChatGPT, Claude 등)를 Action으로 추가해  
> **텍스트(요약 / 분류 / 템플릿 메시지)를 자동 생성**한다.

### 보너스 2 – 실패 알림 및 재시도 전략
> - 워크플로우 실행 중 오류 발생 시 **이메일/메신저로 실패 알림** 전송  
> - **재시도(재실행)** 또는 **대체 경로** 1개 이상 설계  
>   (예: 저장 실패 시 임시 시트에 적재)

### 프로젝트 1 예시 – Make vs Zapier

**워크플로우:**  
`Google Form 제출 → 조건 분기(점수 기준) → Google Sheets 기록 + Slack 알림`

#### Make 구현
Trigger : Google Forms – Watch Responses
Router : 점수 80점 이상 / 미만 분기
Action 1: Google Sheets – Add a Row (합격자 시트)
Action 2: Slack – Send a Message (합격 알림)


#### Zapier 구현
Trigger : Google Forms – New Response in Spreadsheet
Filter : 점수 80점 이상만 통과
Action 1: Google Sheets – Create Spreadsheet Row
Action 2: Slack – Send Channel Message

#### 비교 분석표

| 비교 항목 | Make | Zapier |
|-----------|------|--------|
| UI/UX | 시각적 노드 기반 | 리스트 기반 |
| 설정 난이도 | 초기 학습 곡선 있음 | 직관적 |
| 연동 서비스 범위 | 광범위 (세부 커넥터 차이 존재) | 광범위 (세부 커넥터 차이 존재) |
| 무료 플랜 | 1,000 Ops/월 | 100 Tasks/월 |
| 실행 로그 | 실행 히스토리에서 단계별 데이터 확인 | Task History |

### 프로젝트 2 예시 – 팀 회의록 자동 정리 및 알림

**반복 업무 정의:**  
매주 회의 후 회의록을 Google Docs에 작성하면  
핵심 내용을 요약해 Slack 채널에 공유하는 작업

**선정 도구:** Make  
> 시각적 분기 설계가 직관적이고 무료 Ops 범위 내 구현 가능

**워크플로우 흐름:**
Trigger : Google Docs – Watch Documents (새 문서 감지)
Action 1: 텍스트 파싱 (제목 / 참석자 / 결정사항 추출)
Filter : 결정사항이 존재하는 경우에만 다음 단계로
Action 2: Slack – Send a Message (요약 내용 + 담당자 멘션)
Action 3: Google Sheets – Add a Row (회의록 이력 기록)
